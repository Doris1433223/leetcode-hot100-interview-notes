# Day06 三数之和

## 基础信息

- 题号：15
- 难度：中等
- 核心考点：双指针；围绕双指针收缩/对撞，理解「单次扫描、原地处理、排序后收敛」相关工程选型与复杂度取舍
- 面试考察场景：考察有序结构、边界收缩、原地处理和去重策略。

## 题意分析

在整数数组中找出所有不重复的三元组，使三数之和为 0

核心约束与边界条件：
- 先确认输入规模，再选择能通过的复杂度
- 特别检查空输入、重复值、边界下标、极端值
- 面试讲解时要说明为什么当前解法能覆盖所有合法情况

## 解题方案

### 暴力解法：三重枚举去重

#### 核心思想
用「朴素枚举状态」理解三数之和。直接枚举三元组，重点暴露重复答案问题

#### 实现步骤
1. 按题意直接列出候选状态，例如下标组合、区间、路径或完整中间结果
2. 逐个验证候选是否满足条件，并记录第一个或全部合法答案
3. 用复杂度反推它为什么只适合教学、小数据或离线校验

#### 复杂度分析
- 时间复杂度：O(n³)
- 空间复杂度：O(1)

#### 完整带注释 Python 代码
```python
# 解法说明：三重枚举去重。
def threeSum_bruteforce(nums):
    ans = set()
    n = len(nums)
    for i in range(n):
        for j in range(i + 1, n):
            for k in range(j + 1, n):
                if nums[i] + nums[j] + nums[k] == 0:
                    ans.add(tuple(sorted((nums[i], nums[j], nums[k]))))
    return [list(x) for x in ans]
```

### 最优解法：排序+固定一数+双指针

#### 核心思想
用「边界移动状态」理解三数之和。固定首数后在右侧找两数和，并在三处去重

#### 实现步骤
1. 确定左右指针分别代表的边界，先把窗口或搜索区间初始化清楚
2. 根据题目条件移动其中一个指针，每次移动都必须排除一批不可能答案
3. 在窗口满足或失效的瞬间更新答案，重点检查指针越界和重复元素

#### 复杂度分析
- 时间复杂度：O(n²)
- 空间复杂度：O(1)

#### 完整带注释 Python 代码
```python
# 解法说明：排序+固定一数+双指针。
def threeSum(nums):
    nums.sort()
    ans = []
    n = len(nums)
    for i in range(n - 2):
        if i and nums[i] == nums[i - 1]:
            continue
        left, right = i + 1, n - 1
        while left < right:
            total = nums[i] + nums[left] + nums[right]
            if total == 0:
                ans.append([nums[i], nums[left], nums[right]])
                left += 1
                right -= 1
                while left < right and nums[left] == nums[left - 1]:
                    left += 1
                while left < right and nums[right] == nums[right + 1]:
                    right -= 1
            elif total < 0:
                left += 1
            else:
                right -= 1
    return ans
```

### 拓展变形解法：固定一数+哈希补数

#### 核心思想
用「映射/计数状态」理解三数之和。每轮把两数之和转成哈希查找，适合对比去重成本

#### 实现步骤
1. 设计能唯一表达当前状态的 key，例如数值、频次元组、前缀和或节点引用
2. 扫描输入时先查询历史状态，再决定计数、分组、匹配或累加答案
3. 更新哈希结构时确认顺序，避免把当前元素错误地和自己匹配

#### 复杂度分析
- 时间复杂度：O(n²)
- 空间复杂度：O(n)

#### 完整带注释 Python 代码
```python
# 解法说明：固定一数+哈希补数。
def threeSum_hash(nums):
    nums.sort()
    ans = set()
    n = len(nums)
    for i in range(n - 2):
        seen = set()
        for j in range(i + 1, n):
            need = -nums[i] - nums[j]
            if need in seen:
                ans.add((nums[i], need, nums[j]))
            seen.add(nums[j])
    return [list(x) for x in ans]
```

## 工程&面试思考（重点模块，面向面试官）

### 1. 各解法工程选型取舍

- 小数据或离线校验场景可以使用三重枚举去重，优势是实现直观，便于验证题意和边界。
- 常规面试和线上服务更倾向于排序+固定一数+双指针，需要重点说明复杂度优势、状态维护方式和边界处理。
- 当题目扩展为多结果、批量查询、数据流或约束变化时，可以从固定一数+哈希补数迁移，体现对同类问题的抽象能力。

### 2. 高频面试追问 + 标准回答

Q1：为什么不能只使用最直观的三重枚举去重？

A1：直观解法通常重复扫描或枚举大量候选，在输入规模扩大时延迟不可控。面试中应先说明其教学价值，再给出更稳定的主线方案。

Q2：排序+固定一数+双指针的核心工程收益是什么？

A2：它通过更合适的数据结构或状态维护方式降低复杂度，使处理过程更稳定，适合在高频查询、批处理或在线服务中使用。

Q3：如果题目变形，应该如何调整？

A3：先判断返回目标是否从单解变成多解、从静态输入变成数据流，或是否增加内存限制，再选择固定一数+哈希补数或相邻模型进行扩展。

### 3. 同类题目思路迁移技巧

- 遇到有序数组、区间收缩、去重组合题时，优先尝试左右指针或快慢指针。
- 复盘时不要只记代码模板，应重点记住状态定义、更新条件和终止条件。
- 面试口述建议先给出暴力基线，再说明优化点，最后补充变形场景下的取舍。

## 易错边界复盘

- 确认输入为空、长度为 1 或不满足最小规模时的返回值。
- 注意重复元素、重复答案和去重顺序，避免漏解或重复输出。
- 面试口述时必须说明时间复杂度和空间复杂度的来源。
- 代码实现要与题目要求的返回格式一致，例如返回下标、数量、布尔值或列表。
