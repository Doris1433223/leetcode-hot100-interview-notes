# Day07 接雨水

## 基础信息

- 题号：42
- 难度：困难
- 核心考点：双指针；围绕双指针收缩/对撞，理解「单次扫描、原地处理、排序后收敛」相关工程选型与复杂度取舍
- 面试考察场景：考察有序结构、边界收缩、原地处理和去重策略。

## 题意分析

给定柱状高度，计算下雨后这些柱子之间最多能接住多少雨水

核心约束与边界条件：
- 先确认输入规模，再选择能通过的复杂度
- 特别检查空输入、重复值、边界下标、极端值
- 面试讲解时要说明为什么当前解法能覆盖所有合法情况

## 解题方案

### 暴力解法：按列扫描左右最高

#### 核心思想
用「直接建模」理解接雨水。每根柱子向两边找最高墙，直观但慢

#### 实现步骤
1. 先把输入规模、边界值和返回格式确认清楚
2. 围绕接雨水维护最小必要状态，避免无意义重复扫描
3. 按题目要求返回结果，并单独检查空输入和极端输入

#### 复杂度分析
- 时间复杂度：O(n²)
- 空间复杂度：O(1)

#### 完整带注释 Python 代码
```python
# 解法说明：按列扫描左右最高。
def trap(height):
    left, right = 0, len(height) - 1
    left_max = right_max = 0
    water = 0
    while left < right:
        if height[left] < height[right]:
            left_max = max(left_max, height[left])
            water += left_max - height[left]
            left += 1
        else:
            right_max = max(right_max, height[right])
            water += right_max - height[right]
            right -= 1
    return water
```

### 拓展变形解法：双指针水位收缩

#### 核心思想
用「边界移动状态」理解接雨水。从两端维护左右水位，谁短先结算谁

#### 实现步骤
1. 确定左右指针分别代表的边界，先把窗口或搜索区间初始化清楚
2. 根据题目条件移动其中一个指针，每次移动都必须排除一批不可能答案
3. 在窗口满足或失效的瞬间更新答案，重点检查指针越界和重复元素

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(1)

#### 完整带注释 Python 代码
```python
# 解法说明：双指针水位收缩。
def solution_42_3(items, target=None):
    paired = sorted((x, i) for i, x in enumerate(items))
    left, right = 0, len(paired) - 1
    best = None
    while left < right:
        cur = paired[left][0] + paired[right][0] if target is not None else right - left
        if target is not None and cur == target:
            return [paired[left][1], paired[right][1]]
        if target is not None and cur < target:
            left += 1
        else:
            right -= 1
    return best
```

## 工程&面试思考（重点模块，面向面试官）

### 1. 各解法工程选型取舍

- 小数据或离线校验场景可以使用按列扫描左右最高，优势是实现直观，便于验证题意和边界。
- 常规面试和线上服务更倾向于前后缀最高数组，需要重点说明复杂度优势、状态维护方式和边界处理。
- 当题目扩展为多结果、批量查询、数据流或约束变化时，可以从双指针水位收缩迁移，体现对同类问题的抽象能力。

### 2. 高频面试追问 + 标准回答

Q1：为什么不能只使用最直观的按列扫描左右最高？

A1：直观解法通常重复扫描或枚举大量候选，在输入规模扩大时延迟不可控。面试中应先说明其教学价值，再给出更稳定的主线方案。

Q2：前后缀最高数组的核心工程收益是什么？

A2：它通过更合适的数据结构或状态维护方式降低复杂度，使处理过程更稳定，适合在高频查询、批处理或在线服务中使用。

Q3：如果题目变形，应该如何调整？

A3：先判断返回目标是否从单解变成多解、从静态输入变成数据流，或是否增加内存限制，再选择双指针水位收缩或相邻模型进行扩展。

### 3. 同类题目思路迁移技巧

- 遇到有序数组、区间收缩、去重组合题时，优先尝试左右指针或快慢指针。
- 复盘时不要只记代码模板，应重点记住状态定义、更新条件和终止条件。
- 面试口述建议先给出暴力基线，再说明优化点，最后补充变形场景下的取舍。

## 易错边界复盘

- 确认输入为空、长度为 1 或不满足最小规模时的返回值。
- 注意重复元素、重复答案和去重顺序，避免漏解或重复输出。
- 面试口述时必须说明时间复杂度和空间复杂度的来源。
- 代码实现要与题目要求的返回格式一致，例如返回下标、数量、布尔值或列表。
