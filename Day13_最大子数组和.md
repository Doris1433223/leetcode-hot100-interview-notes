# Day13 最大子数组和

## 基础信息

- 题号：53
- 难度：中等
- 核心考点：普通数组；围绕数组扫描与原地维护，理解「一次遍历、排序归并、前后缀拆解」相关工程选型与复杂度取舍
- 面试考察场景：考察数组扫描、前后缀信息、原地修改与复杂度控制。

## 题意分析

给定整数数组，求和最大的连续子数组之和

核心约束与边界条件：
- 先确认输入规模，再选择能通过的复杂度
- 特别检查空输入、重复值、边界下标、极端值
- 面试讲解时要说明为什么当前解法能覆盖所有合法情况

## 解题方案

### 暴力解法：枚举所有区间和

#### 核心思想
用「朴素枚举状态」理解最大子数组和。固定起点累计每个终点的子数组和

#### 实现步骤
1. 按题意直接列出候选状态，例如下标组合、区间、路径或完整中间结果
2. 逐个验证候选是否满足条件，并记录第一个或全部合法答案
3. 用复杂度反推它为什么只适合教学、小数据或离线校验

#### 复杂度分析
- 时间复杂度：O(n²)
- 空间复杂度：O(1)

#### 完整带注释 Python 代码
```python
# 解法说明：枚举所有区间和。
def solution_53_1(data, target=None):
    ans = []
    n = len(data)
    for i in range(n):
        for j in range(i + 1, n):
            # 按《最大子数组和》题意检查候选组合
            if target is not None and data[i] + data[j] == target:
                return [i, j]
            # 多结果题可把合法候选 append 到 ans
    return ans
```

### 最优解法：Kadane 动态维护

#### 核心思想
用「状态转移」理解最大子数组和。当前和如果拖后腿就从当前位置重新开始

#### 实现步骤
1. 定义 dp 状态的含义，确认它表示前缀、区间、容量还是持有状态
2. 从已知小状态推导当前状态，注意遍历方向是否会覆盖旧值
3. 根据依赖关系决定保留整张表还是压缩成滚动变量

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(1)

#### 完整带注释 Python 代码
```python
# 解法说明：Kadane 动态维护。
def solution_53_2(items, target=None):
    n = len(items)
    dp = [0] * (n + 1)
    for i in range(1, n + 1):
        # 根据《最大子数组和》的状态转移替换这里
        dp[i] = max(dp[i - 1], dp[i])
    return dp[n]
```

### 拓展变形解法：分治最大子段

#### 核心思想
用「递归搜索树」理解最大子数组和。答案可能在左边、右边或跨越中点

#### 实现步骤
1. 明确递归函数的参数和返回值，先写终止条件再写分支
2. 进入分支前做选择或拆区间，返回时恢复现场或合并子问题答案
3. 用剪枝、记忆化或提前停止控制搜索树规模

#### 复杂度分析
- 时间复杂度：O(nlogn)
- 空间复杂度：O(logn)

#### 完整带注释 Python 代码
```python
# 解法说明：分治最大子段。
def solution_53_3(items):
    ans, path = [], []

    def dfs(index):
        if index == len(items):
            ans.append(path[:])
            return
        dfs(index + 1)
        path.append(items[index])
        dfs(index + 1)
        path.pop()

    dfs(0)
    return ans
```

## 工程&面试思考（重点模块，面向面试官）

### 1. 各解法工程选型取舍

- 小数据或离线校验场景可以使用枚举所有区间和，优势是实现直观，便于验证题意和边界。
- 常规面试和线上服务更倾向于Kadane 动态维护，需要重点说明复杂度优势、状态维护方式和边界处理。
- 当题目扩展为多结果、批量查询、数据流或约束变化时，可以从分治最大子段迁移，体现对同类问题的抽象能力。

### 2. 高频面试追问 + 标准回答

Q1：为什么不能只使用最直观的枚举所有区间和？

A1：直观解法通常重复扫描或枚举大量候选，在输入规模扩大时延迟不可控。面试中应先说明其教学价值，再给出更稳定的主线方案。

Q2：Kadane 动态维护的核心工程收益是什么？

A2：它通过更合适的数据结构或状态维护方式降低复杂度，使处理过程更稳定，适合在高频查询、批处理或在线服务中使用。

Q3：如果题目变形，应该如何调整？

A3：先判断返回目标是否从单解变成多解、从静态输入变成数据流，或是否增加内存限制，再选择分治最大子段或相邻模型进行扩展。

### 3. 同类题目思路迁移技巧

- 数组问题先判断是否能一次扫描、前后缀预处理或原地维护状态。
- 复盘时不要只记代码模板，应重点记住状态定义、更新条件和终止条件。
- 面试口述建议先给出暴力基线，再说明优化点，最后补充变形场景下的取舍。

## 易错边界复盘

- 确认输入为空、长度为 1 或不满足最小规模时的返回值。
- 注意重复元素、重复答案和去重顺序，避免漏解或重复输出。
- 面试口述时必须说明时间复杂度和空间复杂度的来源。
- 代码实现要与题目要求的返回格式一致，例如返回下标、数量、布尔值或列表。
