# Day15 轮转数组

## 基础信息

- 题号：189
- 难度：中等
- 核心考点：普通数组；围绕数组扫描与原地维护，理解「一次遍历、排序归并、前后缀拆解」相关工程选型与复杂度取舍
- 面试考察场景：考察数组扫描、前后缀信息、原地修改与复杂度控制。

## 题意分析

给定数组和 k，将数组元素向右轮转 k 个位置

核心约束与边界条件：
- 先确认输入规模，再选择能通过的复杂度
- 特别检查空输入、重复值、边界下标、极端值
- 面试讲解时要说明为什么当前解法能覆盖所有合法情况

## 解题方案

### 暴力解法：额外数组取模放置

#### 核心思想
用「朴素枚举状态」理解轮转数组。每个元素直接放到 (i+k)%n 的新位置

#### 实现步骤
1. 按题意直接列出候选状态，例如下标组合、区间、路径或完整中间结果
2. 逐个验证候选是否满足条件，并记录第一个或全部合法答案
3. 用复杂度反推它为什么只适合教学、小数据或离线校验

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(n)

#### 完整带注释 Python 代码
```python
# 解法说明：额外数组取模放置。
def rotate(nums, k):
    n = len(nums)
    k %= n
    nums.reverse()
    nums[:k] = reversed(nums[:k])
    nums[k:] = reversed(nums[k:])
```

### 最优解法：三次翻转

#### 核心思想
用「直接建模」理解轮转数组。整体翻转、前 k 段翻转、后半段翻转

#### 实现步骤
1. 先把输入规模、边界值和返回格式确认清楚
2. 围绕轮转数组维护最小必要状态，避免无意义重复扫描
3. 按题目要求返回结果，并单独检查空输入和极端输入

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(1)

#### 完整带注释 Python 代码
```python
# 解法说明：三次翻转。
def rotate(nums, k):
    n = len(nums)
    k %= n
    nums.reverse()
    nums[:k] = reversed(nums[:k])
    nums[k:] = reversed(nums[k:])
```

### 拓展变形解法：环状替换

#### 核心思想
用「直接建模」理解轮转数组。沿着下标环搬运元素，记录已移动数量

#### 实现步骤
1. 先把输入规模、边界值和返回格式确认清楚
2. 围绕轮转数组维护最小必要状态，避免无意义重复扫描
3. 按题目要求返回结果，并单独检查空输入和极端输入

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(1)

#### 完整带注释 Python 代码
```python
# 解法说明：环状替换。
def rotate(nums, k):
    n = len(nums)
    k %= n
    nums.reverse()
    nums[:k] = reversed(nums[:k])
    nums[k:] = reversed(nums[k:])
```

## 工程&面试思考（重点模块，面向面试官）

### 1. 各解法工程选型取舍

- 小数据或离线校验场景可以使用额外数组取模放置，优势是实现直观，便于验证题意和边界。
- 常规面试和线上服务更倾向于三次翻转，需要重点说明复杂度优势、状态维护方式和边界处理。
- 当题目扩展为多结果、批量查询、数据流或约束变化时，可以从环状替换迁移，体现对同类问题的抽象能力。

### 2. 高频面试追问 + 标准回答

Q1：为什么不能只使用最直观的额外数组取模放置？

A1：直观解法通常重复扫描或枚举大量候选，在输入规模扩大时延迟不可控。面试中应先说明其教学价值，再给出更稳定的主线方案。

Q2：三次翻转的核心工程收益是什么？

A2：它通过更合适的数据结构或状态维护方式降低复杂度，使处理过程更稳定，适合在高频查询、批处理或在线服务中使用。

Q3：如果题目变形，应该如何调整？

A3：先判断返回目标是否从单解变成多解、从静态输入变成数据流，或是否增加内存限制，再选择环状替换或相邻模型进行扩展。

### 3. 同类题目思路迁移技巧

- 数组问题先判断是否能一次扫描、前后缀预处理或原地维护状态。
- 复盘时不要只记代码模板，应重点记住状态定义、更新条件和终止条件。
- 面试口述建议先给出暴力基线，再说明优化点，最后补充变形场景下的取舍。

## 易错边界复盘

- 确认输入为空、长度为 1 或不满足最小规模时的返回值。
- 注意重复元素、重复答案和去重顺序，避免漏解或重复输出。
- 面试口述时必须说明时间复杂度和空间复杂度的来源。
- 代码实现要与题目要求的返回格式一致，例如返回下标、数量、布尔值或列表。
