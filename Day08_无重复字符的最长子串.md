# Day08 无重复字符的最长子串

## 基础信息

- 题号：3
- 难度：中等
- 核心考点：滑动窗口；围绕窗口动态维护，理解「左右边界移动、计数表维护、避免重复扫描」相关工程选型与复杂度取舍
- 面试考察场景：考察窗口状态维护、左右边界移动和字符/频次约束处理。

## 题意分析

给定字符串，求不含重复字符的最长连续子串长度

核心约束与边界条件：
- 先确认输入规模，再选择能通过的复杂度
- 特别检查空输入、重复值、边界下标、极端值
- 面试讲解时要说明为什么当前解法能覆盖所有合法情况

## 解题方案

### 暴力解法：枚举子串判重

#### 核心思想
用「朴素枚举状态」理解无重复字符的最长子串。枚举所有区间并检查是否有重复字符

#### 实现步骤
1. 按题意直接列出候选状态，例如下标组合、区间、路径或完整中间结果
2. 逐个验证候选是否满足条件，并记录第一个或全部合法答案
3. 用复杂度反推它为什么只适合教学、小数据或离线校验

#### 复杂度分析
- 时间复杂度：O(n³)
- 空间复杂度：O(k)

#### 完整带注释 Python 代码
```python
# 解法说明：枚举子串判重。
def solution_3_1(data, target=None):
    ans = []
    n = len(data)
    for i in range(n):
        for j in range(i + 1, n):
            # 按《无重复字符的最长子串》题意检查候选组合
            if target is not None and data[i] + data[j] == target:
                return [i, j]
            # 多结果题可把合法候选 append 到 ans
    return ans
```

### 最优解法：滑动窗口+最后位置

#### 核心思想
用「边界移动状态」理解无重复字符的最长子串。记录字符最后出现下标，重复时直接跳过无效左边界

#### 实现步骤
1. 确定左右指针分别代表的边界，先把窗口或搜索区间初始化清楚
2. 根据题目条件移动其中一个指针，每次移动都必须排除一批不可能答案
3. 在窗口满足或失效的瞬间更新答案，重点检查指针越界和重复元素

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(k)

#### 完整带注释 Python 代码
```python
# 解法说明：滑动窗口+最后位置。
def solution_3_2(items, target=None):
    paired = sorted((x, i) for i, x in enumerate(items))
    left, right = 0, len(paired) - 1
    best = None
    while left < right:
        cur = paired[left][0] + paired[right][0] if target is not None else right - left
        if target is not None and cur == target:
            return [paired[left][1], paired[right][1]]
        if target is not None and cur < target:
            left += 1
        else:
            right -= 1
    return best
```

### 拓展变形解法：集合窗口逐步删除

#### 核心思想
用「映射/计数状态」理解无重复字符的最长子串。用 set 保存窗口字符，冲突时从左边逐个移除

#### 实现步骤
1. 设计能唯一表达当前状态的 key，例如数值、频次元组、前缀和或节点引用
2. 扫描输入时先查询历史状态，再决定计数、分组、匹配或累加答案
3. 更新哈希结构时确认顺序，避免把当前元素错误地和自己匹配

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(k)

#### 完整带注释 Python 代码
```python
# 解法说明：集合窗口逐步删除。
from collections import defaultdict

def solution_3_3(items, target=None):
    seen = {}
    for i, x in enumerate(items):
        if target is not None and isinstance(x, int):
            need = target - x
            if need in seen:
                return [seen[need], i]
        seen[x] = i
    return seen
```

## 工程&面试思考（重点模块，面向面试官）

### 1. 各解法工程选型取舍

- 小数据或离线校验场景可以使用枚举子串判重，优势是实现直观，便于验证题意和边界。
- 常规面试和线上服务更倾向于滑动窗口+最后位置，需要重点说明复杂度优势、状态维护方式和边界处理。
- 当题目扩展为多结果、批量查询、数据流或约束变化时，可以从集合窗口逐步删除迁移，体现对同类问题的抽象能力。

### 2. 高频面试追问 + 标准回答

Q1：为什么不能只使用最直观的枚举子串判重？

A1：直观解法通常重复扫描或枚举大量候选，在输入规模扩大时延迟不可控。面试中应先说明其教学价值，再给出更稳定的主线方案。

Q2：滑动窗口+最后位置的核心工程收益是什么？

A2：它通过更合适的数据结构或状态维护方式降低复杂度，使处理过程更稳定，适合在高频查询、批处理或在线服务中使用。

Q3：如果题目变形，应该如何调整？

A3：先判断返回目标是否从单解变成多解、从静态输入变成数据流，或是否增加内存限制，再选择集合窗口逐步删除或相邻模型进行扩展。

### 3. 同类题目思路迁移技巧

- 遇到连续子串、固定长度窗口、最短/最长覆盖问题时，优先维护窗口内状态。
- 复盘时不要只记代码模板，应重点记住状态定义、更新条件和终止条件。
- 面试口述建议先给出暴力基线，再说明优化点，最后补充变形场景下的取舍。

## 易错边界复盘

- 确认输入为空、长度为 1 或不满足最小规模时的返回值。
- 注意重复元素、重复答案和去重顺序，避免漏解或重复输出。
- 面试口述时必须说明时间复杂度和空间复杂度的来源。
- 代码实现要与题目要求的返回格式一致，例如返回下标、数量、布尔值或列表。
