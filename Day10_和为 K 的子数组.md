# Day10 和为 K 的子数组

## 基础信息

- 题号：560
- 难度：中等
- 核心考点：子串；围绕前缀和/单调队列/窗口，理解「把连续区间问题转成可复用状态」相关工程选型与复杂度取舍
- 面试考察场景：考察连续区间建模、前缀和、单调结构或窗口约束。

## 题意分析

给定整数数组和整数 k，统计和为 k 的连续子数组个数

核心约束与边界条件：
- 先确认输入规模，再选择能通过的复杂度
- 特别检查空输入、重复值、边界下标、极端值
- 面试讲解时要说明为什么当前解法能覆盖所有合法情况

## 解题方案

### 暴力解法：双层区间求和

#### 核心思想
用「朴素枚举状态」理解和为 K 的子数组。固定左端点向右累加，遇到 k 就计数

#### 实现步骤
1. 按题意直接列出候选状态，例如下标组合、区间、路径或完整中间结果
2. 逐个验证候选是否满足条件，并记录第一个或全部合法答案
3. 用复杂度反推它为什么只适合教学、小数据或离线校验

#### 复杂度分析
- 时间复杂度：O(n²)
- 空间复杂度：O(1)

#### 完整带注释 Python 代码
```python
# 解法说明：双层区间求和。
def solution_560_1(data, target=None):
    ans = []
    n = len(data)
    for i in range(n):
        for j in range(i + 1, n):
            # 按《和为 K 的子数组》题意检查候选组合
            if target is not None and data[i] + data[j] == target:
                return [i, j]
            # 多结果题可把合法候选 append 到 ans
    return ans
```

### 最优解法：前缀和+哈希计数

#### 核心思想
用「映射/计数状态」理解和为 K 的子数组。当前前缀减 k 是否出现过，出现几次就贡献几段

#### 实现步骤
1. 设计能唯一表达当前状态的 key，例如数值、频次元组、前缀和或节点引用
2. 扫描输入时先查询历史状态，再决定计数、分组、匹配或累加答案
3. 更新哈希结构时确认顺序，避免把当前元素错误地和自己匹配

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(n)

#### 完整带注释 Python 代码
```python
# 解法说明：前缀和+哈希计数。
from collections import defaultdict

def solution_560_2(items, target=None):
    seen = {}
    for i, x in enumerate(items):
        if target is not None and isinstance(x, int):
            need = target - x
            if need in seen:
                return [seen[need], i]
        seen[x] = i
    return seen
```

### 拓展变形解法：离线前缀列表统计

#### 核心思想
用「直接建模」理解和为 K 的子数组。收集全部前缀后做排序/计数，适合离线分析场景

#### 实现步骤
1. 先把输入规模、边界值和返回格式确认清楚
2. 围绕和为 K 的子数组维护最小必要状态，避免无意义重复扫描
3. 按题目要求返回结果，并单独检查空输入和极端输入

#### 复杂度分析
- 时间复杂度：O(nlogn)
- 空间复杂度：O(n)

#### 完整带注释 Python 代码
```python
# 解法说明：离线前缀列表统计。
from collections import defaultdict

def subarraySum(nums, k):
    count = defaultdict(int)
    count[0] = 1
    prefix = ans = 0
    for x in nums:
        prefix += x
        ans += count[prefix - k]
        count[prefix] += 1
    return ans
```

## 工程&面试思考（重点模块，面向面试官）

### 1. 各解法工程选型取舍

- 小数据或离线校验场景可以使用双层区间求和，优势是实现直观，便于验证题意和边界。
- 常规面试和线上服务更倾向于前缀和+哈希计数，需要重点说明复杂度优势、状态维护方式和边界处理。
- 当题目扩展为多结果、批量查询、数据流或约束变化时，可以从离线前缀列表统计迁移，体现对同类问题的抽象能力。

### 2. 高频面试追问 + 标准回答

Q1：为什么不能只使用最直观的双层区间求和？

A1：直观解法通常重复扫描或枚举大量候选，在输入规模扩大时延迟不可控。面试中应先说明其教学价值，再给出更稳定的主线方案。

Q2：前缀和+哈希计数的核心工程收益是什么？

A2：它通过更合适的数据结构或状态维护方式降低复杂度，使处理过程更稳定，适合在高频查询、批处理或在线服务中使用。

Q3：如果题目变形，应该如何调整？

A3：先判断返回目标是否从单解变成多解、从静态输入变成数据流，或是否增加内存限制，再选择离线前缀列表统计或相邻模型进行扩展。

### 3. 同类题目思路迁移技巧

- 连续区间求和或计数问题通常可以迁移到前缀和、哈希计数或单调队列。
- 复盘时不要只记代码模板，应重点记住状态定义、更新条件和终止条件。
- 面试口述建议先给出暴力基线，再说明优化点，最后补充变形场景下的取舍。

## 易错边界复盘

- 确认输入为空、长度为 1 或不满足最小规模时的返回值。
- 注意重复元素、重复答案和去重顺序，避免漏解或重复输出。
- 面试口述时必须说明时间复杂度和空间复杂度的来源。
- 代码实现要与题目要求的返回格式一致，例如返回下标、数量、布尔值或列表。
