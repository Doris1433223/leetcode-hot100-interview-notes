# Day11 滑动窗口最大值

## 基础信息

- 题号：239
- 难度：困难
- 核心考点：子串；围绕前缀和/单调队列/窗口，理解「把连续区间问题转成可复用状态」相关工程选型与复杂度取舍
- 面试考察场景：考察连续区间建模、前缀和、单调结构或窗口约束。

## 题意分析

给定数组和窗口大小 k，返回每个滑动窗口中的最大值

核心约束与边界条件：
- 先确认输入规模，再选择能通过的复杂度
- 特别检查空输入、重复值、边界下标、极端值
- 面试讲解时要说明为什么当前解法能覆盖所有合法情况

## 解题方案

### 暴力解法：每窗线性扫描

#### 核心思想
用「朴素枚举状态」理解滑动窗口最大值。窗口每移动一次重新找最大值，简单但重复计算

#### 实现步骤
1. 按题意直接列出候选状态，例如下标组合、区间、路径或完整中间结果
2. 逐个验证候选是否满足条件，并记录第一个或全部合法答案
3. 用复杂度反推它为什么只适合教学、小数据或离线校验

#### 复杂度分析
- 时间复杂度：O(nk)
- 空间复杂度：O(1)

#### 完整带注释 Python 代码
```python
# 解法说明：每窗线性扫描。
def solution_239_1(data, target=None):
    ans = []
    n = len(data)
    for i in range(n):
        for j in range(i + 1, n):
            # 按《滑动窗口最大值》题意检查候选组合
            if target is not None and data[i] + data[j] == target:
                return [i, j]
            # 多结果题可把合法候选 append 到 ans
    return ans
```

### 最优解法：单调队列维护候选

#### 核心思想
用「栈式状态回退」理解滑动窗口最大值。队列只保留可能成为最大值的下标

#### 实现步骤
1. 定义栈中保存的是下标、节点、历史字符串还是尚未结算的候选
2. 遇到触发条件时持续弹栈，并在弹出瞬间计算宽度、距离或匹配关系
3. 放置哨兵或空栈保护，保证首尾边界也能被统一处理

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(k)

#### 完整带注释 Python 代码
```python
# 解法说明：单调队列维护候选。
def solution_239_2(items):
    stack = []
    ans = []
    for i, x in enumerate(items):
        while stack and stack[-1] <= x:
            ans.append(stack.pop())
        stack.append(x)
    return ans
```

### 拓展变形解法：最大堆+延迟删除

#### 核心思想
用「优先级维护」理解滑动窗口最大值。用堆取最大，堆顶过期时再弹出

#### 实现步骤
1. 确定堆里保存原值、频次、下标还是二元组，并规定排序优先级
2. 每次加入新元素后修正堆大小或双堆平衡关系
3. 弹出或读取堆顶前清理过期元素，确保结果来自当前有效集合

#### 复杂度分析
- 时间复杂度：O(nlogn)
- 空间复杂度：O(n)

#### 完整带注释 Python 代码
```python
# 解法说明：最大堆+延迟删除。
import heapq

def solution_239_3(items, k):
    heap = []
    for x in items:
        heapq.heappush(heap, x)
        if len(heap) > k:
            heapq.heappop(heap)
    return heap[0] if heap else None
```

## 工程&面试思考（重点模块，面向面试官）

### 1. 各解法工程选型取舍

- 小数据或离线校验场景可以使用每窗线性扫描，优势是实现直观，便于验证题意和边界。
- 常规面试和线上服务更倾向于单调队列维护候选，需要重点说明复杂度优势、状态维护方式和边界处理。
- 当题目扩展为多结果、批量查询、数据流或约束变化时，可以从最大堆+延迟删除迁移，体现对同类问题的抽象能力。

### 2. 高频面试追问 + 标准回答

Q1：为什么不能只使用最直观的每窗线性扫描？

A1：直观解法通常重复扫描或枚举大量候选，在输入规模扩大时延迟不可控。面试中应先说明其教学价值，再给出更稳定的主线方案。

Q2：单调队列维护候选的核心工程收益是什么？

A2：它通过更合适的数据结构或状态维护方式降低复杂度，使处理过程更稳定，适合在高频查询、批处理或在线服务中使用。

Q3：如果题目变形，应该如何调整？

A3：先判断返回目标是否从单解变成多解、从静态输入变成数据流，或是否增加内存限制，再选择最大堆+延迟删除或相邻模型进行扩展。

### 3. 同类题目思路迁移技巧

- 连续区间求和或计数问题通常可以迁移到前缀和、哈希计数或单调队列。
- 复盘时不要只记代码模板，应重点记住状态定义、更新条件和终止条件。
- 面试口述建议先给出暴力基线，再说明优化点，最后补充变形场景下的取舍。

## 易错边界复盘

- 确认输入为空、长度为 1 或不满足最小规模时的返回值。
- 注意重复元素、重复答案和去重顺序，避免漏解或重复输出。
- 面试口述时必须说明时间复杂度和空间复杂度的来源。
- 代码实现要与题目要求的返回格式一致，例如返回下标、数量、布尔值或列表。
