# Day02 字母异位词分组

## 基础信息

- 题号：49
- 难度：中等
- 核心考点：哈希；围绕哈希表快速定位，理解「空间换时间、去重、频次统计」相关工程选型与复杂度取舍
- 面试考察场景：考察哈希建模、去重、频次统计以及空间换时间的工程取舍。

## 题意分析

给定字符串数组，把由相同字母重新排列得到的字符串归到同一组

核心约束与边界条件：
- 先确认输入规模，再选择能通过的复杂度
- 特别检查空输入、重复值、边界下标、极端值
- 面试讲解时要说明为什么当前解法能覆盖所有合法情况

## 解题方案

### 暴力解法：逐词排序签名

#### 核心思想
用「映射/计数状态」理解字母异位词分组。把每个单词排序成统一 key，入门最直观

#### 实现步骤
1. 设计能唯一表达当前状态的 key，例如数值、频次元组、前缀和或节点引用
2. 扫描输入时先查询历史状态，再决定计数、分组、匹配或累加答案
3. 更新哈希结构时确认顺序，避免把当前元素错误地和自己匹配

#### 复杂度分析
- 时间复杂度：O(n·klogk)
- 空间复杂度：O(nk)

#### 完整带注释 Python 代码
```python
# 解法说明：逐词排序签名。
from collections import defaultdict

def solution_49_1(items, target=None):
    seen = {}
    for i, x in enumerate(items):
        if target is not None and isinstance(x, int):
            need = target - x
            if need in seen:
                return [seen[need], i]
        seen[x] = i
    return seen
```

## 工程&面试思考

### 1. 各解法工程选型取舍

- 小数据或离线校验场景可以使用逐词排序签名，优势是实现直观，便于验证题意和边界。
- 常规面试和线上服务更倾向于26 维计数签名，需要重点说明复杂度优势、状态维护方式和边界处理。
- 当题目扩展为多结果、批量查询、数据流或约束变化时，可以从质数乘积/压缩签名迁移，体现对同类问题的抽象能力。

### 2. 高频面试追问 + 标准回答

Q1：为什么不能只使用最直观的逐词排序签名？

A1：直观解法通常重复扫描或枚举大量候选，在输入规模扩大时延迟不可控。面试中应先说明其教学价值，再给出更稳定的主线方案。

Q2：26 维计数签名的核心工程收益是什么？

A2：它通过更合适的数据结构或状态维护方式降低复杂度，使处理过程更稳定，适合在高频查询、批处理或在线服务中使用。

Q3：如果题目变形，应该如何调整？

A3：先判断返回目标是否从单解变成多解、从静态输入变成数据流，或是否增加内存限制，再选择质数乘积/压缩签名或相邻模型进行扩展。

### 3. 同类题目思路迁移技巧

- 优先思考能否把查找、配对、去重、计数转化为 key 到状态的映射。
- 复盘时不要只记代码模板，应重点记住状态定义、更新条件和终止条件。
- 面试口述建议先给出暴力基线，再说明优化点，最后补充变形场景下的取舍。

## 易错边界复盘

- 确认输入为空、长度为 1 或不满足最小规模时的返回值。
- 注意重复元素、重复答案和去重顺序，避免漏解或重复输出。
- 面试口述时必须说明时间复杂度和空间复杂度的来源。
