# Day04 移动零

## 基础信息

- 题号：283
- 难度：简单
- 核心考点：双指针；围绕快慢指针稳定压缩，理解「单次扫描、原地处理、相对顺序」相关工程选型与复杂度取舍
- 面试考察场景：考察数组原地修改、快慢指针建模、稳定顺序维护和空间优化策略。

## 题意分析

给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。
请注意 ，必须在不复制数组的情况下原地对数组进行操作。

核心约束与边界条件：
- 先确认输入规模，再选择能通过的复杂度
- 特别检查空数组、全 0、无 0、非零元素相对顺序等情况
- 面试讲解时要说明为什么当前解法是原地修改，而不是返回新数组

## 解题方案

### 暴力解法：额外数组重写

#### 核心思想
用「额外数组暂存」理解移动零。先收集所有非零元素，再把 0 补到末尾，最容易保证相对顺序。

#### 实现步骤
1. 遍历数组，先把所有非零元素收集到额外数组中
2. 将非零元素按原顺序覆盖回 nums 前半部分
3. 从非零元素结束位置开始，把 nums 剩余位置全部补 0
4. 这种方式逻辑直观，但使用了额外空间，不满足严格原地要求

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(n)

#### 完整带注释 Python 代码
```python
# 解法说明：额外数组重写。
class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        # 收集全部非零元素
        non_zero = [x for x in nums if x != 0]
        # 先把非零覆盖到数组前面
        for i in range(len(non_zero)):
            nums[i] = non_zero[i]
        # 末尾补0
        for i in range(len(non_zero), len(nums)):
            nums[i] = 0
```

### 最优解法：快慢指针稳定压缩

#### 核心思想
用「快慢指针压缩」理解移动零。slow 指向下一个非零元素应该放置的位置，fast 负责扫描整个数组。

#### 实现步骤
1. 初始化 slow，表示下一个非零元素的落点
2. 使用 fast 从左到右扫描数组
3. 遇到非零元素时，将它覆盖到 slow 位置，并让 slow 后移
4. 扫描结束后，slow 左侧已经按原顺序放好了全部非零元素
5. 将 slow 之后的位置全部填充为 0，完成原地移动

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(1)

#### 完整带注释 Python 代码
```python
# 解法说明：快慢指针稳定压缩。
class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        slow = 0
        # fast扫描所有数字，非零放到slow位置
        for fast in range(len(nums)):
            if nums[fast] != 0:
                nums[slow] = nums[fast]
                slow += 1
        # slow之后全部填充0
        for i in range(slow, len(nums)):
            nums[i] = 0
```

### 拓展变形解法：交换式原地搬运

#### 核心思想
用「交换式快慢指针」理解移动零。遇到非零元素就和 slow 位置交换，让非零元素逐步移动到数组前面。

#### 实现步骤
1. 初始化 slow，表示下一个非零元素应该交换到的位置
2. 使用 fast 扫描数组，寻找非零元素
3. 每遇到一个非零元素，就将 nums[fast] 和 nums[slow] 交换
4. 交换后 slow 后移，继续维护前半部分都是已处理的非零元素
5. 整个过程只需要一次扫描，不需要额外数组和末尾补零

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(1)

#### 完整带注释 Python 代码
```python
# 解法说明：交换式原地搬运。
class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        slow = 0
        for fast in range(len(nums)):
            if nums[fast] != 0:
                # 交换快慢指针位置
                nums[slow], nums[fast] = nums[fast], nums[slow]
                slow += 1
```

## 工程&面试思考

### 1. 各解法工程选型取舍

- 额外数组重写适合帮助理解题意，核心是先保留非零元素顺序，再统一补零。
- 常规面试更推荐快慢指针稳定压缩，因为它满足原地修改、稳定顺序和 O(1) 空间。
- 交换式原地搬运可以作为拓展方案，用来说明同样是快慢指针，也可以通过交换维护数组状态。

### 2. 高频面试追问 + 标准回答

Q1：为什么不能只使用额外数组重写？

A1：题目明确要求原地操作，额外数组虽然简单，但空间复杂度是 O(n)，不符合严格约束。

Q2：快慢指针稳定压缩的核心工程收益是什么？

A2：它把数组前半部分维护成已经处理好的非零区间，只用一次扫描和常数空间完成稳定移动。

Q3：为什么这种写法能保持非零元素的相对顺序？

A3：fast 是从左到右扫描的，非零元素会按原出现顺序依次写入 slow 位置，因此相对顺序不会改变。

### 3. 同类题目思路迁移技巧

- 遇到原地数组整理问题，优先思考快慢指针是否能维护一个已处理区间。
- slow 通常表示下一个合法元素的落点，fast 负责扫描和筛选。
- 面试中先说明额外数组基线，再说明如何把额外空间优化成原地覆盖。

## 易错边界复盘

- 空数组和长度为 1 的数组不需要特殊修改。
- 全 0 数组中 slow 不会移动，最后仍然全部填 0。
- 无 0 数组中非零元素会按原顺序保留。
- 题目要求修改原数组，不需要返回新数组。
- 快慢指针覆盖写法最后必须把 slow 后面补 0。
- 交换式写法可以省掉补零，但要理解 slow 前面维护的是非零区间。
