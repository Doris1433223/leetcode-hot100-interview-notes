# Day05 盛最多水的容器

## 基础信息

- 题号：11
- 难度：中等
- 核心考点：双指针；围绕左右边界对撞，理解「面积瓶颈、短板移动、单次扫描」相关工程选型与复杂度取舍
- 面试考察场景：考察双指针收缩、状态剪枝、面积计算和复杂度优化策略。

## 题意分析

给定一个长度为 n 的整数数组 height 。有 n 条垂线，第 i 条线的两个端点是 (i, 0) 和 (i, height[i]) 。
找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。
返回容器可以储存的最大水量。

说明：你不能倾斜容器。

核心约束与边界条件：
- 先确认输入规模，再选择能通过的复杂度
- 特别检查数组长度、左右边界、相同高度、单调高度等情况
- 面试讲解时要说明为什么每次移动短板不会漏掉最优答案

## 解题方案

### 暴力解法：暴力两线枚举

#### 核心思想
用「枚举左右边界」理解盛最多水的容器。任意两条线都可以组成一个容器，面积由宽度和较短高度共同决定。

#### 实现步骤
1. 枚举每一个左边界
2. 在左边界右侧枚举每一个右边界
3. 用两边较短高度乘以下标距离，得到当前容器面积
4. 扫描所有边界组合，维护最大面积
5. 该方法覆盖所有可能答案，但组合数量较多

#### 复杂度分析
- 时间复杂度：O(n²)
- 空间复杂度：O(1)

#### 完整带注释 Python 代码
```python
# 解法说明：暴力两线枚举。
class Solution:
    def maxArea(self, height: List[int]) -> int:
        max_water = 0
        n = len(height)
        # 枚举左边界
        for left in range(n):
            # 枚举右边界（在左侧右边）
            for right in range(left + 1, n):
                # 容器高度由短板决定
                h = min(height[left], height[right])
                w = right - left
                area = h * w
                if area > max_water:
                    max_water = area
        return max_water
```

### 最优解法：对撞双指针

#### 核心思想
用「左右边界收缩」理解盛最多水的容器。面积高度受短板限制，固定短板时继续缩小宽度不可能得到更优答案，因此每次移动短板。

#### 实现步骤
1. 初始化 left 指向数组开头，right 指向数组末尾
2. 根据当前左右边界计算容器面积，并更新最大值
3. 比较左右高度，移动高度较小的一侧
4. 因为宽度一定变小，只有尝试更高的短板才可能扩大面积
5. 当左右指针相遇时，所有有效边界组合已完成剪枝判断

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(1)

#### 完整带注释 Python 代码
```python
# 解法说明：对撞双指针。
class Solution:
    def maxArea(self, height: List[int]) -> int:
        max_water = 0
        left = 0
        right = len(height) - 1
        while left < right:
            h = min(height[left], height[right])
            w = right - left
            max_water = max(max_water, h * w)
            # 移动短板
            if height[left] < height[right]:
                left += 1
            else:
                right -= 1
        return max_water
```

### 拓展变形解法：高柱剪枝扫描

#### 核心思想
用「候选边界压缩」理解盛最多水的容器。被更外侧且更高的柱子支配的边界不可能成为最优解，可以先筛出有效高柱候选。

#### 实现步骤
1. 从左向右筛选高度不断变高的左边界候选
2. 从右向左筛选高度不断变高的右边界候选
3. 左右候选分别保留不被外侧高柱支配的位置
4. 在候选高柱之间使用类似双指针的方式计算面积
5. 每次仍然移动短板一侧，继续尝试更高边界

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(n)

#### 完整带注释 Python 代码
```python
# 解法说明：高柱剪枝扫描。
class Solution:
    def maxArea(self, height: List[int]) -> int:
        n = len(height)
        max_water = 0
        # 筛选左侧递增高柱候选
        left_candidates = []
        max_h = 0
        for i in range(n):
            if height[i] > max_h:
                left_candidates.append((i, height[i]))
                max_h = height[i]
        # 筛选右侧递增高柱候选
        right_candidates = []
        max_h = 0
        for i in range(n-1, -1, -1):
            if height[i] > max_h:
                right_candidates.append((i, height[i]))
                max_h = height[i]
        # 双指针遍历候选高柱
        l = r = 0
        while l < len(left_candidates) and r < len(right_candidates):
            li, lh = left_candidates[l]
            ri, rh = right_candidates[r]
            if li >= ri:
                break
            area = min(lh, rh) * (ri - li)
            max_water = max(max_water, area)
            # 移动短板
            if lh < rh:
                l += 1
            else:
                r += 1
        return max_water
```

## 方案横向对比

| 类型 | 方案 | 时间复杂度 | 空间复杂度 | 优缺点 | 适用场景 |
| --- | --- | --- | --- | --- | --- |
| 暴力解法 | 暴力两线枚举 | O(n²) | O(1) | 覆盖所有组合，最容易理解；但枚举成本高，数据量大时不可用 | 入门理解、验证面积公式和边界含义 |
| 最优解法 | 对撞双指针 | O(n) | O(1) | 单次扫描，空间最优；关键是要讲清楚为什么移动短板 | 常规面试主推解法，适合静态数组求最大面积 |
| 拓展变形解法 | 高柱剪枝扫描 | O(n) | O(n) | 先压缩候选边界，再做短板移动；但实现更绕，额外空间更高 | 拓展讨论，适合解释支配关系和候选剪枝 |

## 工程&面试思考

### 1. 各解法工程选型取舍

- 暴力两线枚举适合作为基线方案，能直接说明面积由左右边界和短板高度决定。
- 常规面试更推荐对撞双指针，因为它用 O(1) 空间和 O(n) 时间完成全局最优搜索。
- 高柱剪枝扫描适合拓展讨论，重点在于说明哪些边界会被外侧更高柱子支配，从而可以提前排除。

### 2. 高频面试追问 + 标准回答

Q1：为什么不能只使用暴力两线枚举？

A1：暴力枚举需要检查所有左右边界组合，时间复杂度是 O(n²)，输入规模变大时会超时。

Q2：对撞双指针为什么每次移动短板？

A2：当前面积受短板限制，如果移动长板，宽度变小而短板不变，面积不可能变大；只有移动短板，才可能遇到更高边界来弥补宽度减少。

Q3：如果左右高度相等，移动哪一边？

A3：移动任意一边都可以。因为当前高度已经由两边共同限制，宽度继续缩小时，只有寻找更高边界才有机会得到更大面积。

### 3. 同类题目思路迁移技巧

- 遇到“两端边界共同决定答案”的数组题，可以优先考虑左右指针对撞。
- 关键不是机械移动指针，而是说明每次移动能排除哪些不可能更优的状态。
- 面试中先给出暴力枚举，再用短板效应解释为什么可以降到线性扫描。

## 易错边界复盘

- 容器宽度是 `right - left`，不是柱子数量。
- 容器高度由较短的柱子决定。
- 每次更新答案后再移动指针，避免漏算当前边界。
- 移动的是短板一侧，不是高度更高的一侧。
- 数组长度小于 2 时无法形成容器，结果保持为 0。
- 高度相等时移动任意一边都不影响正确性。
