# Day03 最长连续序列

## 基础信息

- 题号：128
- 难度：中等
- 核心考点：哈希；围绕哈希表快速定位，理解「空间换时间、去重、频次统计」相关工程选型与复杂度取舍
- 面试考察场景：考察哈希建模、去重、连续状态判断以及空间换时间的工程取舍。

## 题意分析

给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

核心约束与边界条件：
- 先确认输入规模，再选择能通过的复杂度
- 特别检查空输入、重复值、负数、断档区间等情况
- 面试讲解时要说明为什么当前解法不会重复统计同一段连续序列

## 解题方案

### 暴力解法：排序后连续扫描

#### 核心思想
用「有序化处理」理解最长连续序列。先排序，让连续数字变成相邻关系，再跳过重复值统计连续段长度。

#### 实现步骤
1. 先处理空数组边界，避免后续初始化连续长度出错
2. 对数组排序，使原本无序的数字转化为有序相邻关系
3. 从左到右扫描排序后的数组，遇到重复数字直接跳过
4. 如果当前数字等于前一个数字加一，说明连续段延长
5. 如果出现断档，则重新开始统计当前连续段
6. 扫描过程中维护最大连续长度，最后返回答案

#### 复杂度分析
- 时间复杂度：O(nlogn)
- 空间复杂度：O(1) 或 O(n)

#### 完整带注释 Python 代码
```python
# 解法说明：排序后连续扫描。
class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        # 边界：空数组直接返回0
        if not nums:
            return 0
        nums.sort()
        max_len = 1   
        cur_len = 1   
        for i in range(1, len(nums)):
            if nums[i] == nums[i - 1]:
                continue
            elif nums[i] == nums[i - 1] + 1:
                cur_len += 1
            # 出现断档，重置当前连续长度
            else:
                cur_len = 1
            max_len = max(max_len, cur_len)
        return max_len
```

### 最优解法：哈希集合起点扩展

#### 核心思想
用「哈希集合快速查找」理解最长连续序列。只从没有前驱的数字开始扩展，保证每段连续序列只被统计一次。

#### 实现步骤
1. 将数组转成哈希集合，去重的同时支持 O(1) 查询某个数字是否存在
2. 遍历集合中的每个数字，判断它是否没有前驱 `num - 1`
3. 只有没有前驱的数字，才可能是一段连续序列的起点
4. 从起点开始不断向后查找 `cur_num + 1`，统计当前连续长度
5. 每完成一段连续序列的扩展，就更新最大长度
6. 遍历结束后返回最大连续序列长度

#### 复杂度分析
- 时间复杂度：O(n)
- 空间复杂度：O(n)

#### 完整带注释 Python 代码
```python
# 解法说明：哈希集合起点扩展。
class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        num_set = set(nums)
        max_len = 0
        for num in num_set:
            # 判断是否为连续序列起点
            if num - 1 not in num_set:
                cur_num = num
                cur_len = 1
                # 向后查找连续数字
                while cur_num + 1 in num_set:
                    cur_num += 1
                    cur_len += 1
                max_len = max(max_len, cur_len)
        return max_len
```

### 拓展变形解法：并查集合并相邻值

#### 核心思想
用「连通分量」理解最长连续序列。把相邻数字看成同一集合，通过合并相邻值维护每个连续块的大小。

#### 实现步骤
1. 用哈希表初始化并查集，每个不同数字先自成一个集合
2. 使用 `parent` 记录父节点，使用 `size` 记录根节点对应集合大小
3. 遍历数组中的数字，如果 `num - 1` 存在，就合并这两个相邻数字
4. 合并时通过路径压缩和按集合大小合并，降低查询与合并成本
5. 每次合并后，根据当前数字所在集合大小更新最大连续长度
6. 遍历结束后返回最大集合大小，即最长连续序列长度

#### 复杂度分析
- 时间复杂度：O(nα(n))
- 空间复杂度：O(n)

#### 完整带注释 Python 代码
```python
class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        parent = dict()   # 存储每个数字的父节点
        size = dict()     # 记录每个根节点对应的集合大小
        max_len = 0

        # 初始化并查集：每个数字自成一个集合，大小为1
        for num in nums:
            if num not in parent:
                parent[num] = num
                size[num] = 1
        
        # 查找根节点（带路径压缩）
        def find(x):
            if parent[x] != x:
                parent[x] = find(parent[x])
            return parent[x]
        
        # 合并两个数字所在集合
        def union(x, y):
            x_root = find(x)
            y_root = find(y)
            if x_root == y_root:
                return
            # 小集合合并到大集合，更新根节点size
            if size[x_root] < size[y_root]:
                x_root, y_root = y_root, x_root
            parent[y_root] = x_root
            size[x_root] += size[y_root]
        
        # 遍历数字，和左侧相邻数合并
        for num in nums:
            if num - 1 in parent:
                union(num, num - 1)
            # 更新当前最大长度
            max_len = max(max_len, size[find(num)])
        
        return max_len
```

## 方案横向对比

| 类型 | 方案 | 时间复杂度 | 空间复杂度 | 优缺点 | 适用场景 |
| --- | --- | --- | --- | --- | --- |
| 暴力解法 | 排序后连续扫描 | O(nlogn) | O(1) 或 O(n) | 思路直观，边界容易处理；但排序不满足线性时间要求 | 小数据、快速验证、对空间更敏感的场景 |
| 最优解法 | 哈希集合起点扩展 | O(n) | O(n) | 通过起点判断避免重复扩展，整体线性；但需要额外哈希集合空间 | 常规面试主推解法，适合静态数组求最长长度 |
| 拓展变形解法 | 并查集合并相邻值 | O(nα(n)) | O(n) | 能表达动态合并关系，但实现更复杂，常规静态题不如哈希集合简洁 | 拓展讨论、动态加入数字、连通关系建模场景 |

## 工程&面试思考

### 1. 各解法工程选型取舍

- 小数据或先快速写出可读解法时，可以使用排序后连续扫描，重点处理重复数字和断档重置。
- 常规面试更推荐哈希集合起点扩展，因为它满足题目要求的线性时间复杂度，并且逻辑足够简洁。
- 并查集适合用于拓展变形，尤其是把相邻数字抽象成连通关系，或者需要支持动态合并时。

### 2. 高频面试追问 + 标准回答

Q1：为什么不能只使用最直观的排序后连续扫描？

A1：排序后扫描虽然简单，但排序本身需要 O(nlogn)，如果题目要求 O(n) 时间复杂度，就需要用哈希集合优化查找过程。

Q2：哈希集合起点扩展的核心工程收益是什么？

A2：它只从没有前驱的数字开始扩展，每个连续段只会被完整扫描一次，避免从段内每个数字重复向后查找。

Q3：如果题目变形，应该如何调整？

A3：如果只返回长度，哈希集合足够；如果要返回具体序列，可以额外记录起点和长度；如果数字动态加入，可以考虑并查集维护连续块大小。

### 3. 同类题目思路迁移技巧

- 遇到“连续”“相邻”“存在性判断”，优先考虑哈希集合降低查找成本。
- 关键不是从每个数字都暴力扩展，而是找到不会重复计算的起点。
- 面试中先给排序基线，再说明哈希优化如何把 O(nlogn) 降到 O(n)。

## 易错边界复盘

- 空数组应返回 0。
- 重复数字不能重复计入连续长度。
- 连续序列不要求在原数组中相邻，只看数值是否连续。
- 哈希集合解法必须只从 `num - 1` 不存在的数字开始扩展。
- 并查集初始化时要对重复数字去重，避免集合大小被重复累加。
- 面试口述时要说明为什么哈希集合整体不是 O(n²)，而是每段序列只从起点扫描一次。
